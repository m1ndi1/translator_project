# Документация программы TheBestTranslator

## 1. Общая информация

**TheBestTranslator** — это веб-приложение для перевода текста и перевода текста с изображений. Программа объединяет в одном интерфейсе две прикладные задачи:

- перевод обычного текста между русским и английским языками;
- распознавание текста на изображении с последующим переводом.

Приложение реализовано по клиент-серверной архитектуре. Пользователь взаимодействует с web-интерфейсом, а backend обрабатывает запросы, выполняет OCR, обращается к сервису перевода и возвращает результат.

## 2. Назначение программы

Программа предназначена для автоматизации перевода коротких текстов и текста, содержащегося на изображениях. Она может использоваться:

- для быстрого перевода фраз и предложений;
- для обработки текста со скриншотов, фотографий и изображений;
- как учебный проект, демонстрирующий работу web-приложения с OCR и внешним API.

Основная идея проекта заключается в том, чтобы пользователь мог выполнить перевод без необходимости вручную перепечатывать текст с изображения.

## 3. Основные возможности

Программа поддерживает следующие функции:

- перевод текста `ru -> en` и `en -> ru`;
- распознавание текста на изображениях;
- перевод распознанного текста;
- загрузку изображений форматов `PNG` и `JPEG`;
- проверку корректности файла перед обработкой;
- встроенную документацию API через FastAPI.

## 4. Архитектура приложения

Проект состоит из двух основных частей:

- **frontend** — пользовательский интерфейс;
- **backend** — серверная часть и API.

### 4.1. Frontend

Frontend находится в директории `frontend/` и отвечает за взаимодействие с пользователем.

Он реализует:

- ввод текста для перевода;
- выбор исходного и целевого языка;
- переключение между режимами работы с текстом и изображением;
- загрузку изображения через выбор файла или drag-and-drop;
- отображение результата перевода;
- вывод статуса обработки;
- переключение темы интерфейса.

Frontend отправляет запросы в backend по префиксу `/api`.

### 4.2. Backend

Backend находится в директории `backend/app/` и реализован на FastAPI.

Он отвечает за:

- прием HTTP-запросов;
- проверку корректности данных;
- распознавание текста на изображениях;
- взаимодействие с Yandex Translate API;
- формирование ответов в формате JSON;
- отдачу frontend-страницы;
- генерацию OpenAPI-документации.

## 5. Структура проекта

- `backend/app/main.py` — точка входа FastAPI-приложения.
- `backend/app/config.py` — конфигурация через переменные окружения.
- `backend/app/routers/` — HTTP-маршруты API.
- `backend/app/services/` — бизнес-логика OCR и перевода.

## 6. Принцип работы программы

### 6.1. Перевод текста

Сценарий работы:

1. Пользователь вводит текст в интерфейсе.
2. Выбирает исходный и целевой язык.
3. Frontend отправляет `POST` запрос на `/api/translate/text`.
4. Backend проверяет корректность входных данных.
5. Backend отправляет запрос в Yandex Translate API.
6. Полученный перевод возвращается пользователю.

### 6.2. OCR изображения

Сценарий работы:

1. Пользователь загружает изображение.
2. Backend проверяет тип файла и его размер.
3. Изображение сохраняется во временную папку.
4. Файл открывается с помощью Pillow.
5. Выполняется базовая предобработка изображения:
   - перевод в оттенки серого;
   - повышение контрастности.
6. Tesseract OCR распознает текст.
7. Временный файл удаляется.
8. Распознанный текст возвращается клиенту.

### 6.3. Перевод изображения

Этот режим объединяет два предыдущих:

1. Загружается изображение.
2. Выполняется OCR.
3. Распознанный текст передается в сервис перевода.
4. Пользователь получает уже готовый перевод.

## 8. Внешние зависимости

Для полной работы программы используются:

- **Tesseract OCR** — для распознавания текста;
- **Yandex Translate API** — для перевода текста.

Если Tesseract не установлен или Yandex API не настроен, соответствующая часть функциональности работать не будет.

## 9. Поддерживаемые форматы и ограничения

На текущий момент программа имеет следующие ограничения:

- поддерживаются только языки `ru` и `en`;
- поддерживаются только изображения `PNG` и `JPEG`;
- максимальный размер изображения по умолчанию — `10 МБ`;
- качество OCR зависит от качества исходного изображения;
- приложение не хранит историю переводов;
- в проекте не реализована авторизация пользователей.

## 10. Конфигурация приложения

Настройки задаются через файл `.env` в каталоге `backend/`.

Пример:

```env
YANDEX_FOLDER_ID=your_folder_id
YANDEX_API_KEY=your_api_key
TESSERACT_CMD=tesseract
TESSERACT_LANG=rus+eng
MAX_IMAGE_SIZE_MB=10
TEMP_DIR=temp
APP_NAME=TheBest Translator API
APP_VERSION=0.1.0
DEBUG=true
API_PREFIX=/api
```

### Назначение параметров

- `YANDEX_FOLDER_ID` — идентификатор каталога Yandex Cloud;
- `YANDEX_API_KEY` — ключ доступа к сервису перевода;
- `TESSERACT_CMD` — путь к исполняемому файлу `tesseract`;
- `TESSERACT_LANG` — набор языков для OCR;
- `MAX_IMAGE_SIZE_MB` — максимальный размер изображения;
- `TEMP_DIR` — директория временных файлов;
- `API_PREFIX` — префикс API.

## 11. Запуск программы

### 11.1. Локальный запуск

Из корня проекта:

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r backend\requirements.txt
cd backend
uvicorn app.main:app --reload
```

После запуска:

- `http://127.0.0.1:8000/` — web-интерфейс;
- `http://127.0.0.1:8000/docs` — Swagger UI;
- `http://127.0.0.1:8000/redoc` — ReDoc.

### 11.2. Запуск через Docker

Сборка:

```powershell
docker build -t thebesttranslator .
```

Запуск:

```powershell
docker run --rm -p 8000:8000 `
  -e YANDEX_FOLDER_ID=your_folder_id `
  -e YANDEX_API_KEY=your_api_key `
  thebesttranslator
```

## 12. Описание API

Базовый префикс API: `/api`

### 12.1. Проверка состояния сервиса

**GET** `/api/health`

Возвращает:

```json
{
  "status": "ok"
}
```

### 12.2. Перевод текста

**POST** `/api/translate/text`

Тело запроса:

```json
{
  "text": "Привет, мир",
  "source_language": "ru",
  "target_language": "en"
}
```

Ответ:

```json
{
  "translated_text": "Hello, world",
  "source_language": "ru",
  "target_language": "en"
}
```

### 12.3. Распознавание текста на изображении

**POST** `/api/ocr/image`

Формат: `multipart/form-data`

Параметры:

- `file` — изображение `PNG` или `JPEG`.

Ответ:

```json
{
  "recognized_text": "Распознанный текст"
}
```

### 12.4. Перевод изображения

**POST** `/api/translate/image`

Формат: `multipart/form-data`

Параметры:

- `file` — изображение `PNG` или `JPEG`;
- `source_language` — исходный язык;
- `target_language` — целевой язык.

Ответ:

```json
{
  "recognized_text": "Hello",
  "translated_text": "Привет",
  "source_language": "en",
  "target_language": "ru"
}
```

## 13. Обработка ошибок

Программа возвращает:

- `400 Bad Request` — некорректный язык, пустой текст, неподдерживаемый файл, слишком большой файл;
- `500 Internal Server Error` — проблемы с OCR, отсутствует Tesseract, не настроен Yandex API, ошибка внешнего сервиса.
